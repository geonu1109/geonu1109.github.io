똑똑하게 시간 데이터 관리하기

3계층 구조 기반의 예시와 함께하는 시간 정보의 생성, 연산, 저장에 관한 고찰



1. 시간

 시간이라는 단어는 여러가지 의미를 가지고 있다. 여러 시간의 의미 중 데이터로서 가치를 가지는 세 가지 의미와 이를 설명하기 위한 토대가 되는 기초적인 의미에 대해 짚고 넘어가보자. 먼저 기초적으로 시간은 순간순간의 흐름을 말한다. 이 글에서 앞으로 이 시간은 세월이라고 한다. 다음으로 데이터로서 의미를 가지는 세 가지 시간을 알아보자. 첫 번 째로 시간은 하루 중 특정 시점을 나타내는 데 사용된다. 이를 앞으로 시각이라고 한다. 시각은 일반적으로 시, 분, 초로 표현되며 알림 시간, 출근 시간 등을 표현하는 데 사용된다. 두 번 째로 시간은 세월의 한 시점을 말한다. 이를 앞으로 시점이라고 한다. 시점은 주로 날짜와 시각의 합으로 표현되며 접속 시간 등을 나타내는 데 사용된다. 마지막으로 시간은 특정 시점과 다른 시점 사이의 기간을 뜻한다. 이를 앞으로 기간이라고 한다. 기간은 일 수와 걸린 시간의 합으로 표현되며 근무 시간, 주행 시간 등을 나타내는 데 사용된다. 이러한 명칭 등은 분류를 위해 이 글에서 임의로 부여해 사용한다.



2. 시각

 시각 정보를 초 단위의 정밀도를 갖도록 다루는 방법에 대해 살펴보자. 시각 정보는 일반적으로 1. 문자열, 2. 시, 분, 초를 변수로 갖는 객체, 3. 하루의 시작으로부터 경과한 초 등으로 표현된다. 첫 째로, 문자열로 시각을 표현하는 방법은 여러가지가 있겠으나 이 글에서 문자열로 다루는 시간은 ISO 8601을 채택한 경우만 고려한다. 이 경우 시각 정보는 콜론으로 시, 분, 초를 각각 구분한다. 예를 들어 오전 여덟시 정각은 08:00:00으로 표현된다. 둘 째로, 시, 분, 초를 변수로 갖는 객체를 만드는 경우는 아래와 같은 자바코드로 클래스를 만들 수 있다.

```java
public class Time {
  private int hour;
  private int minute;
  private int second;
}
```

마지막으로 하루의 시작으로부터 경과한 초로 표기하는 경우 오전 여덟시 정각은 자정에서 여덟시간이 지났기 때문에 이를 초로 환산하면 8 * 60 * 60이 되어 28,000이 된다.



 시각 정보를 문자열 포멧으로 다루는 경우 최대 장점은 사람이 읽기 편하다는 것이다. 특히 GUI 환경에서 시각을 표현할 필요가 있는 경우에는 기존에 어떤 포멧을 사용했더라도 반드시 문자열 포멧으로 변환하는 과정이 필요하다.

 시각 정보를 하나의 객체로 다루는 경우의 장점은 시, 분, 초 각각을 독립적으로 활용할 수 있게 된다. 이는 시각을 다루는 데에 높은 자유도를 부여한다. 예를 들면 여덟시를 08:00:00에서 오전 8시로 손 쉽게 표기를 전환할 수 있도록 한다.

 시각 정보를 하루 중 경과한 초로 표현하는 방법은 연산에 매우 유리하다. 예를 들어 여덟시에서 90초가 경과한 경우 28,000에 90만 더하여 28,080으로 수정하면 그만이다. 단위에 따른 변화를 고려하지 않아도 된다. 또한 정렬 등을 위한 대소 비교시에도 조건을 복잡하게 만들 필요가 없다.



 세가지 방법을 모두 소개한 후 제안하는 시각 정보 표현 방법은 다음과 같다. 시각을 표현하는 객체를 만들되, 내부적으로 시각 정보는 경과한 초로 나타내고 시, 분, 초를 나타내는 메서드와 문자열 변환 메서드를 제공한다.

```java
public class Time {
  private int timestamp;
  
  public int getHour() {
    return this.timestamp / 60 / 60;
  }
  
  public int getMinute() {
    return this.timestamp % (60 * 60) / 60;
  }
  
  public int getSecond() {
    return this.timestamp % 60;
  }
  
  public String toString() {
    return this.getHour().toString() + ":" + this.getMinute().toString() + ":" + this.getSecond().toString();
  }
}
```

위와 같이 객체를 만들 경우, 프로그램 내에서 자유롭게 연산이 가능한 반면 화면 출력을 위한 문자열 변환은 간편해지고 데이터 저장시에도 최소한의 용량만 사용하게 된다.
